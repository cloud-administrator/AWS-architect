# 最短設定手順.md

## 最短ルート

会社PCで、次の順番で進めます。自宅PCでは実接続しません。

## 1. Snowflake側でMCP用roleを作る

```sql
USE ROLE SECURITYADMIN;

CREATE ROLE IF NOT EXISTS <MCP_ADMIN_ROLE>;
CREATE ROLE IF NOT EXISTS <MCP_ACCESS_ROLE>;

GRANT ROLE <MCP_ACCESS_ROLE> TO USER <USERNAME>;
GRANT ROLE <MCP_ADMIN_ROLE> TO USER <ADMIN_USERNAME>;
```

## 2. 必要権限を付与する

```sql
USE ROLE ACCOUNTADMIN;

GRANT USAGE ON DATABASE <DATABASE> TO ROLE <MCP_ADMIN_ROLE>;
GRANT USAGE ON SCHEMA <DATABASE>.<SCHEMA> TO ROLE <MCP_ADMIN_ROLE>;
GRANT CREATE MCP SERVER ON SCHEMA <DATABASE>.<SCHEMA> TO ROLE <MCP_ADMIN_ROLE>;

GRANT USAGE ON DATABASE <DATABASE> TO ROLE <MCP_ACCESS_ROLE>;
GRANT USAGE ON SCHEMA <DATABASE>.<SCHEMA> TO ROLE <MCP_ACCESS_ROLE>;
GRANT USAGE ON WAREHOUSE <WAREHOUSE> TO ROLE <MCP_ACCESS_ROLE>;

GRANT SELECT ON SEMANTIC VIEW <DATABASE>.<SCHEMA>.<SEMANTIC_VIEW>
  TO ROLE <MCP_ACCESS_ROLE>;

GRANT USAGE ON CORTEX SEARCH SERVICE <DATABASE>.<SCHEMA>.<SEARCH_SERVICE>
  TO ROLE <MCP_ACCESS_ROLE>;

GRANT USAGE ON MCP SERVER <DATABASE>.<SCHEMA>.<MCP_SERVER_NAME>
  TO ROLE <MCP_ACCESS_ROLE>;
```

最後の`GRANT USAGE ON MCP SERVER`は、MCP server作成後に実行します。先に実行して失敗した場合は、作成後に再実行してください。

## 3. 接続ユーザーのdefaultを設定する

```sql
ALTER USER <USERNAME>
  SET DEFAULT_ROLE = '<MCP_ACCESS_ROLE>'
      DEFAULT_WAREHOUSE = '<WAREHOUSE>';
```

## 4. MCP serverを作る

```sql
USE ROLE <MCP_ADMIN_ROLE>;
USE DATABASE <DATABASE>;
USE SCHEMA <SCHEMA>;

CREATE OR REPLACE MCP SERVER <MCP_SERVER_NAME>
  FROM SPECIFICATION $$
    tools:
      - name: "business-analyst"
        type: "CORTEX_ANALYST_MESSAGE"
        identifier: "<DATABASE>.<SCHEMA>.<SEMANTIC_VIEW>"
        title: "Business Analyst"
        description: "Governed semantic analysis for approved business data."

      - name: "document-search"
        type: "CORTEX_SEARCH_SERVICE_QUERY"
        identifier: "<DATABASE>.<SCHEMA>.<SEARCH_SERVICE>"
        title: "Document Search"
        description: "Searches approved business documents through Cortex Search."
  $$;

GRANT USAGE ON MCP SERVER <DATABASE>.<SCHEMA>.<MCP_SERVER_NAME>
  TO ROLE <MCP_ACCESS_ROLE>;
```

## 5. Snowflake OAuth integrationを作る

```sql
USE ROLE ACCOUNTADMIN;

CREATE OR REPLACE SECURITY INTEGRATION <OAUTH_INTEGRATION_NAME>
  TYPE = OAUTH
  OAUTH_CLIENT = CUSTOM
  ENABLED = TRUE
  OAUTH_CLIENT_TYPE = 'CONFIDENTIAL'
  OAUTH_REDIRECT_URI = '<CODEX_REDIRECT_URI>'
  OAUTH_ISSUE_REFRESH_TOKENS = TRUE
  OAUTH_REFRESH_TOKEN_VALIDITY = 86400
  OAUTH_USE_SECONDARY_ROLES = NONE
  BLOCKED_ROLES_LIST = ('ACCOUNTADMIN', 'SECURITYADMIN', 'ORGADMIN', 'GLOBALORGADMIN');

SELECT SYSTEM$SHOW_OAUTH_CLIENT_SECRETS('<OAUTH_INTEGRATION_NAME_IN_UPPERCASE>');
```

`client_id`と`client_secret`は会社のsecret管理ルールに従って保管します。

## 6. Private LinkとDNSを確認する

Azure Private Link構成後、会社PCで確認します。

```powershell
Resolve-DnsName <ORG>-<ACCOUNT>.privatelink.snowflakecomputing.com
Test-NetConnection <ORG>-<ACCOUNT>.privatelink.snowflakecomputing.com -Port 443
```

Snowflake側:

```sql
SELECT SYSTEM$ALLOWLIST_PRIVATELINK();
```

## 7. CodexにMCP serverを登録する

```powershell
codex mcp add snowflake-private `
  --url "https://<ORG>-<ACCOUNT>.privatelink.snowflakecomputing.com/api/v2/databases/<DATABASE>/schemas/<SCHEMA>/mcp-servers/<MCP_SERVER_NAME>" `
  --oauth-client-id "<OAUTH_CLIENT_ID>"
```

## 8. OAuthログインする

```powershell
codex mcp login snowflake-private
```

## 9. Codexで確認する

```powershell
codex mcp list
codex mcp get snowflake-private --json
```

Codex画面内では次を実行します。

```text
/mcp
```

## 10. 試験質問を実行する

例:

```text
Snowflake MCPのBusiness Analyst toolを使って、売上の概要を確認してください。
```

確認観点:

- Snowflake MCP serverが見える。
- 想定したtoolだけが見える。
- 権限不足の場合は安全に失敗する。
- Snowflake query history / access historyで実行者を追跡できる。
