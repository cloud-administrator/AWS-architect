## 結論

営業支援型のClaude Enterpriseでは、現行の通常モデルなら **「シート料金」＋「従量課金」** です。シート料金はClaudeへのアクセス権の料金で、利用量は含まれません。Claude、Claude Code、Coworkで消費したトークンは、標準API料金で別途課金されます。営業支援型Enterpriseでは、その利用分が月次で後払い請求されます。([Claudeヘルプセンター][1])

管理者コンソールで行うべき制御は、厳密には「クレジット購入量の制御」ではなく、**Spend limits（利用金額上限）** の設定です。Sales-assisted Enterpriseではクレジット残高を消費する方式ではなく、全員の利用が月次請求に集計されるため、上限を設けたい場合はSpend limitsを設定します。([Claudeヘルプセンター][1])

## 設定方法：営業支援型Enterpriseの場合

前提として、設定できるのは通常 **OwnerまたはPrimary Owner** です。グループ単位の上限は、Billing権限を持つカスタムロールでも管理できます。([Claudeヘルプセンター][1])

### 1. 管理者でClaudeにログインする

OwnerまたはPrimary OwnerでClaudeにログインします。

### 2. 管理画面を開く

**Organization settings → Usage** に移動します。Anthropicの公式説明では、EnterpriseのSpend limitsはこの画面から設定します。([Claudeヘルプセンター][1])

### 3. 組織全体の上限を設定する

Organization levelのSpend limitを設定すると、組織全体の利用額の上限になります。これは全ユーザー・全利用面に対するハード上限なので、到達すると組織全体に影響します。Anthropicは、組織全体の上限は慎重に使い、まずはグループ単位・ユーザー単位の上限から始めることを推奨しています。([Claudeヘルプセンター][2])

### 4. グループ単位の上限を設定する

中〜大規模展開では、グループ単位が実務上扱いやすいです。

手順は以下です。

1. **Organization settings → Usage** に移動
2. **By group** タブを選択
3. 対象グループの右側メニューから **Edit limit** を選択
4. **Set dollar amount** で月額上限を入力、または **Unlimited** を選択
5. **Set limit** をクリック

このグループ上限は、そのグループの各メンバーに対する **1人あたり月額上限** として適用されます。([Claudeヘルプセンター][3])

### 5. ユーザー単位の上限を設定する

個別ユーザーにSpend limitを設定できます。個別上限はグループ上限より優先されます。たとえば、営業部は一律$30/月、特定のPoC担当者だけ$100/月、開発者は$300/月のような設計が可能です。([Claudeヘルプセンター][3])

### 6. 複数グループ所属時のルールを設定する

ユーザーが複数グループに所属している場合、**Multi-group spend limit** で「高い方の上限を使う」か「低い方の上限を使う」かを選べます。場所は **Organization settings → Usage → Spending defaults** です。([Claudeヘルプセンター][3])

### 7. 上限到達時の挙動を理解しておく

営業支援型Enterpriseでは、Spend limitに到達すると、その上限に該当する利用は次の請求期間まで停止します。OwnerまたはPrimary Ownerが上限を引き上げれば再開できます。なお、上限を「Unlimited」にしても従量課金は止まりません。usage-based pricingはEnterpriseの中核仕様です。([Claudeヘルプセンター][1])

## 契約形態の整理

### 現行のEnterprise契約

公開情報上、現行Enterpriseは **1種類のEnterpriseシート** をユーザー単位・月額で契約し、年次請求される形です。このシート料金にClaudeの利用量は含まれず、Claude、Claude Code、Coworkの利用は標準API料金で別課金されます。([Claudeヘルプセンター][4])

つまり、質問の表現に合わせると、**「シートを買うだけで基本料金がかかり、それとは別に、使った分だけ従量課金される」** という理解で概ね正しいです。

### 営業支援型Enterprise

営業支援型Enterpriseでは、利用分はクレジット残高から引かれるのではなく、**月次で後払い請求** されます。誰かが多く使っても他の人の残高を枯渇させるわけではありませんが、その分、月次請求額が増えます。したがって、予算統制にはSpend limitsを設定します。([Claudeヘルプセンター][1])

### Self-serve Enterpriseとの違い

Self-serve Enterpriseでは、利用分を前払いクレジットとして購入し、組織全体で共有する方式です。クレジットがなくなると利用が止まります。一方、営業支援型Enterpriseでは残高の概念ではなく、実消費が月次請求に載ります。([Claudeヘルプセンター][1])

### レガシーのStandard/Premium席の場合だけ注意

もし管理画面の **Organization settings → Organization** やMembers画面で **Standard / Premium** の席種が見える場合、それは古いseat-based Enterpriseです。この場合は、シート料金に一定の利用枠が含まれ、上限到達後に **Usage credits** を使って継続利用できます。OwnerまたはPrimary Ownerが **Organization settings → Usage → Usage credits → Enable** から有効化し、組織全体・席種別・ユーザー別の月額上限を設定します。([Claudeヘルプセンター][1])

ただしAnthropicは、Standard/Premiumのseat-based Enterpriseは新規契約では利用できないレガシー方式で、次回更新時にusage-basedの単一Enterpriseシートへ移行すると説明しています。([Claudeヘルプセンター][1])

## 導入時の推奨設定

営業支援型Enterpriseなら、初期設定は次の順が安全です。

1. 組織全体の上限は高め、または一旦未設定
2. 部門・用途別にグループを作成
3. グループごとに月額上限を設定
4. Claude CodeやCoworkのような高消費機能は対象グループに段階展開
5. 例外ユーザーだけ個別上限で上書き
6. 月次で **Organization settings → Usage** と請求書を確認

Anthropicも、Claude CodeやCoworkは通常のチャットよりトークン消費が大きくなりやすく、アクセス制御・グループ上限・ユーザー上限を組み合わせる運用を推奨しています。([Claudeヘルプセンター][2])
