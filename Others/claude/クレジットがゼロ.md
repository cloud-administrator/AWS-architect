# クレジットがゼロ

> GitHub表示用にDOCXからMarkdown化しています。

Anthropic公式情報ベースでは、「クレジットがゼロになったらそれ以上使わせない」設定は、セルフサービスEnterpriseでは基本的に標準挙動です。別途「ゼロ時に停止」トグルを有効化するというより、自動補充を使わず、必要に応じて支出制限を設定する運用になります。

## 1. セルフサービスEnterpriseの場合

公式ヘルプでは、セルフサービスEnterpriseは使用量をクレジットで事前購入し、Claude / Claude Code の利用に応じてクレジットが減少すると説明されています。クレジットがなくなると、Owner または Primary Owner が追加購入するまで使用は停止します。(Claudeヘルプセンター)

設定・確認手順は次の通りです。

Owner または Primary Owner で Claude にログイン

Organization settings > Usage に移動

クレジット残高を確認

追加購入を自動化したくない場合は、クレジットカードによる自動リロードを設定しない

さらに上限を設ける場合は、同じ Organization settings > Usage で支出制限を設定する

公式ヘルプでは、セルフサービスEnterpriseでは全ユーザーが同じクレジット残高から消費し、残高がゼロになると組織全体の使用が停止すると明記されています。(Claudeヘルプセンター)

## 2. 支出制限も設定する場合

Organization settings > Usage で、Owner / Primary Owner が支出制限を設定できます。設定できるレベルは公式上、少なくとも以下です。

組織レベル: 組織全体の最大支出

個別ユーザーレベル: 特定ユーザーの最大支出

これらは階層的に動作し、ユーザーは「個別制限」または「組織制限」の低い方を超えられません。セルフサービスプランでは、クレジット残高が残っていても支出制限に達すると使用が停止します。(Claudeヘルプセンター)

## 3. 営業支援Enterpriseの場合は別

営業支援Enterpriseは、クレジットを事前購入する方式ではなく、使用量が月次で後払い請求されます。この場合、ゼロになるクレジット残高はありません。上限をかけたい場合は、Organization settings > Usage で支出制限を設定します。支出制限に達すると、次の請求期間が始まるか、Owner が制限を引き上げるまで使用が停止します。(Claudeヘルプセンター)

## 4. グループ単位で制御したい場合

Enterpriseではグループ別の支出制限も利用できます。公式ヘルプでは、Organization settings > Groups でグループを管理し、Organization settings > Usage でグループ支出制限を管理できるとされています。グループ支出制限は「グループ内の各メンバーに対するユーザーあたり月間支出制限」です。(Claudeヘルプセンター)

実務上は、全社停止リスクを避けるために、まず部門・ロール別のグループ支出制限とユーザー個別制限を設定し、最後に組織全体のハード上限を設定する構成が安全です。公式のEnterprise利用ガイドでも、支出制限を「消費制御の主要手段」とし、組織・グループ・個別ユーザーの3段階で制御できると説明されています。(Claudeヘルプセンター)

## 5. レガシーのシートベースEnterpriseの場合

Standard / Premium シートがある古いEnterpriseでは、「追加使用量」を有効にすると、シートに含まれる使用量制限に達した後も Claude / Cowork / Claude Code を使い続けられます。逆に、制限到達後に使わせたくない場合は、追加使用量を有効にしない、または無効化する運用になります。公式ヘルプでは、追加使用量を完全に無効化でき、その場合メンバーは使用量制限に達すると作業を続けられず、制限リセットまで待つ必要があるとされています。(Claudeヘルプセンター)

結論として、今回の要件が「Enterprise導入で、クレジット残高がゼロになったら止める」なら、セルフサービスEnterpriseでは標準で止まります。管理者設定としては、Owner / Primary Owner で Organization settings > Usage を開き、クレジット残高・自動リロード有無・組織/ユーザー/グループ支出制限を確認・設定するのが該当手順です。
