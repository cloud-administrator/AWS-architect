# モデル学習に使われないようにする設定

> GitHub表示用にDOCXからMarkdown化しています。

Anthropic公式情報だけで確認した結論です。

## 1. モデル学習に使われないようにする設定

Claude Enterprise / Claude for Work では、入力・出力はデフォルトでモデル学習に使われません。 Anthropic公式のPrivacy Centerは、商用製品の入力・出力は「デフォルトではモデル学習に使わない」と明記しています。ただし、ユーザーが thumbs up/down などでフィードバックを送信した場合、または組織が明示的にデータ利用を許可した場合は例外になり得ます。(privacy.anthropic.com)

管理者コンソールで実務上やるべき設定は、フィードバック送信を無効化することです。

手順

Primary Owner または Owner で Claude にログイン

Organization settings > Data and Privacy を開く

Rate chats のトグルをオフにする

Anthropic公式ヘルプでは、Team / Enterprise の Primary Owner または Owner が、組織メンバーによる thumbs up/down フィードバック送信可否を管理できると説明されています。(Claudeヘルプセンター) また、フィードバックを送ると関連する会話全体が最大5年間保存され、Anthropicがモデル訓練を含む目的で利用し得るとされています。(privacy.anthropic.com)

加えて、Development Partner Program に参加していないことも確認してください。Anthropicは、商用製品のデータはデフォルトでは学習に使わない一方、この任意プログラムでは Claude Code セッションの共有を通じてモデル訓練を含む改善に使われ得ると説明しています。プリペイド請求のClaude Consoleでは Settings > Privacy controls > Development Partner Program > Leave で退出できます。請求書払いの組織はアカウント担当に確認する扱いです。(Claudeヘルプセンター)

## 2. 日本リージョンだけで処理されるようにする設定

Claude Enterprise の管理コンソールでは、日本リージョン固定の設定は、公式ドキュメント上は確認できません。  
現時点で公式ヘルプに記載されている Enterprise 組織向けのリージョン制御は US-only inference です。これは推論、つまり Claude が回答を生成するモデル処理を米国内サーバーに限定する設定です。(Claudeヘルプセンター)

US-only inference の設定手順

Primary Owner / Owner、または Privacy 権限が “Can manage” のカスタムロールでログイン

Organization settings > Data and privacy を開く

US-only inference を探す

トグルをオンにする

この設定は組織全体に適用され、Claude apps、Claude Code、Claude Desktop、会話タイトルやmemoryなどのバックグラウンド処理にも適用されます。ただし、利用可能なのは usage-based Enterprise plan で、Team plan や legacy seat-based Enterprise では表示されません。(Claudeヘルプセンター)

重要な制約として、US-only inference は推論処理のみを対象にし、Slack や Google Drive などのコネクタ／第三者サービスの処理場所や、データ保存場所は制御しません。(Claudeヘルプセンター)

Claude API / Claude Console 側でも、Anthropic公式の Data residency ドキュメント上、inference_geo で指定できる値は "global" と "us" のみで、Workspace geo も現在は "us" のみです。公式ドキュメントの “Current limitations” でも、Inference geo は "us" と "global" のみ、Workspace geo は "us" のみとされています。(Claude)

なお、Anthropicの Amazon Bedrock 向け公式ドキュメントには、Bedrock の Regional endpoints / CRIS は US、EU、Japan、Asia-Pacific で利用可能と記載されています。ただし、これは Claude Enterprise 管理コンソールの設定ではなく、Amazon Bedrock 経由でClaudeを使う場合の構成です。(Claude Platform Docs)

## 導入案件での回答方針

顧客向けには、次のように説明するのが正確です。

学習利用について  
Claude Enterprise の商用データはデフォルトでモデル学習に使われない。追加統制として、管理者が Organization settings > Data and Privacy > Rate chats をオフにし、Development Partner Program に参加していないことを確認する。

日本リージョン限定について  
Claude Enterprise の管理コンソールには、日本リージョン固定の設定は公式情報上確認できない。Enterprise 管理コンソールで提供されている地域制御は US-only inference。日本国内処理が必須要件であれば、Claude Enterprise アプリではなく、Amazon Bedrock の日本リージョン／Regional endpoint 構成など、別アーキテクチャとして検討が必要。
