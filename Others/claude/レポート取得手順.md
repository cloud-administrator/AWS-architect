# レポート取得手順

> GitHub表示用にDOCXからMarkdown化しています。

以下は、Anthropic / Claude公式サイトのみを根拠にした、Claude Enterprise導入時の「レポート取得手順」です。確認日は 2026年6月30日 です。

## まず結論：用途別に取得方法が違います

| 取得したいもの | 主な取得場所 | 主な権限 | 出力・特徴 |
| --- | --- | --- | --- |
| 利用・費用レポート | Claude.ai の Analytics > Spend | Owner / Primary Owner。Enterprise Admin は Analytics を見られるが Spend は除外 | CSV。ユーザー別・モデル別の利用量と推定/実績コスト |
| Claude Code 利用レポート | Analytics > Claude Code | Enterprise の Owner / Primary Owner / Admin など | 画面確認。ユーザー別の一部データは CSV Export 可能 |
| 監査ログ | Organization settings > Data and privacy > Export logs | Organization Owner / Primary Owner | 過去180日分。メールでダウンロードリンク。リンクは24時間有効 |
| 組織データ export | Organization settings > Data and privacy > Export Data | Primary Owner のみ | 会話データ・ユーザーデータ。メールでダウンロードリンク |
| 継続的なBI/監査連携 | Analytics API / Compliance API | Primary Owner が API キー作成 | APIで取得。用途によりキーとスコープが異なる |

Claude Enterprise には、監査ログ、Compliance API、Analytics API が含まれると公式に説明されています。Analytics API は集計済みのエンゲージメント/採用指標、Compliance API はアクティビティログ・チャット履歴・ファイル内容などへのプログラムアクセス用途です。(Claudeヘルプセンター)

## 1. 利用・費用レポート CSV を取得する手順

対象は、利用量・トークン・モデル別費用・ユーザー別費用を確認したい場合です。

手順は次の通りです。

Owner または Primary Owner のアカウントで Claude にログインします。  
公式情報では、Enterprise の Usage analytics は Owner / Primary Owner / Admin が利用できますが、Enterprise Admin は Spend を除く Analytics までとされています。(Claudeヘルプセンター)

左下の自分のイニシャルをクリックし、Analytics を選択します。  
公式ヘルプでは、Primary Owner と Owner は左下のイニシャルから Analytics にアクセスできると説明されています。(Claudeヘルプセンター)

Settings > Analytics に移動します。

Spend セクションまでスクロールします。

Export Spend Report をクリックします。

期間を選択します。選べる期間は、MTD、Last Month、Last 90 Days、Custom です。Custom の場合は開始日と終了日を指定します。公式情報では、Custom は最大90日前まで遡れ、取得可能な最新データは「昨日」までとされています。(Claudeヘルプセンター)

Download をクリックして CSV を取得します。(Claudeヘルプセンター)

CSV には、ユーザーのメールアドレス、Account UUID、Product、Model / Model family、リクエスト数、Prompt tokens、Completion tokens、Net spend、Gross spend などが含まれます。Net spend は割引・クレジット・契約レート適用後の実コスト、Gross spend は割引前コストです。(Claudeヘルプセンター)

注意点として、Spend データは日次更新で 1日遅延 があります。Usage-based Enterprise では組織の全利用が対象ですが、Seat-based Enterprise で usage credits が有効な場合は、seat allotment を超えた利用分のみが反映されます。(Claudeヘルプセンター)

## 2. Analytics 画面で確認できる主な指標

All activity では、Weekly Active Users、利用率、保留中の招待、Claude / Claude Code / Cowork 別の Daily・Weekly・Monthly Active Users、Top connectors などを確認できます。Spend セクションでは、MTD/QTD/YTD の総支出、モデル別支出、支出上位10ユーザーが確認できます。(Claudeヘルプセンター)

Claude.ai の個別 Analytics では、Chats per day、1回以上チャットしたユーザー比率、チャット総数、Projects、Artifacts などの指標が確認できます。Claude Code と Cowork には、それぞれ製品別の Analytics 画面があります。(Claudeヘルプセンター)

不明点：公式ページ上で、Spend Report 以外の全 Analytics 指標を一括 CSV export できるかは確認できませんでした。公式に明確な手順がある CSV export は、少なくとも Spend Report と Claude Code の一部 Export です。

## 3. Claude Code 利用レポートを取得する手順

Claude Code の利用状況を確認したい場合は、次の手順です。

Owner または Primary Owner のアカウントでログインします。Enterprise では Owner / Primary Owner / Admin が Claude Code usage analytics を利用できますが、usage-based plan では Chat + Claude Code seat、seat-based plan では Premium seat が必要とされています。(Claude ヘルプセンター)

左下のイニシャルまたは名前をクリックします。

Analytics > Claude Code に移動します。(Claude ヘルプセンター)

Claude Code Usage 画面で、組織レベル指標を確認します。確認できる指標には、Lines of code accepted、Suggestion accept rate、Active users / sessions の日次推移、accepted code lines の日次推移などがあります。(Claude ヘルプセンター)

ユーザー別の利用状況では、各メンバーのメールアドレスと当月の total lines of code accepted を確認できます。この画面では Export ボタンから、メンバーのメールアドレスと total lines of code を含む CSV を生成できます。(Claude ヘルプセンター)

Claude Code の利用指標は現在の暦月分を表示し、月初にリセットされます。過去データの可視化は日次粒度で確認できます。(Claude ヘルプセンター)

## 4. 監査ログを取得する手順

セキュリティ監査・操作履歴・管理者操作の確認が目的なら、監査ログを取得します。

Organization Owner または Primary Owner でログインします。監査ログは Enterprise 組織のみ利用可能です。(Claude ヘルプセンター)

Organization settings > Data and Privacy に移動します。

Export logs ボタンをクリックします。

実行すると、組織の過去 180日分 の監査ログが集約されます。

リクエストした Owner 宛に、ダウンロードリンク付きのメールが届きます。リンクは 24時間有効 です。ログ集約のため、メール到着まで遅延が発生する場合があります。(Claude ヘルプセンター)

監査ログには、created_at、actor_info、event、event_info、entity_info、ip_address、device_id、user_agent、client_platform などの列が含まれます。(Claude ヘルプセンター)

重要な制限として、監査ログにはチャットやプロジェクトのタイトル・本文は含まれず、ユニークIDのみが export されます。チャットの入出力は、Primary Owner による data export 側で export 可能とされています。(Claude ヘルプセンター)

また、Enterprise plan で customer-managed encryption keys を使っている組織では、Export logs ボタンでは監査ログを export できません。この場合、監査ログイベントは Compliance API で利用可能とされています。(Claude ヘルプセンター)

## 5. 組織データを export する手順

会話データやユーザーデータを export したい場合は、監査ログではなく Organization data export を使います。

Primary Owner のアカウントでログインします。Team / Enterprise の組織データ export は Primary Owner のみ利用可能で、個別メンバーにはセルフサービス export オプションはありません。(Claudeヘルプセンター)

左下のイニシャルをクリックします。

Organization settings を選択します。

Data and privacy を開きます。

Export Data をクリックします。

export 処理後、アカウントに紐づくメールアドレス宛にダウンロードリンクが届きます。(Claudeヘルプセンター)

注意点として、export 開始前にユーザーが手動削除した、または enterprise retention settings により削除された Messages / files / projects は、export には含まれません。メール内のリンクからデータをダウンロードするにはアカウントにサインインしている必要があり、リンクは配信後24時間で期限切れになります。(Claudeヘルプセンター)

不明点：公式ページ上では、この organization data export のファイル形式、スキーマ、列定義、文字コード、具体的な全項目は確認できませんでした。

## 6. APIでレポートを継続取得する手順

BIツール、FinOps、SIEM、監査基盤に連携する場合は、画面のCSVではなく API を使うのが適しています。

### 6-1. Claude Enterprise Analytics API

用途は、組織全体のエンゲージメント、採用状況、利用量、費用をプログラムで取得することです。Claude Enterprise Analytics API は Claude Enterprise 組織向けで、engagement / adoption data は全 Enterprise plan、cost / usage endpoints は usage-based Enterprise plan に適用されます。Seat-based Enterprise plan では usage credits 分が反映されます。(Claude)

手順は次の通りです。

Primary Owner で Claude にサインインします。公式ドキュメントでは、Analytics API access の有効化と Analytics API key 作成は Primary Owner のみ可能とされています。(Claude)

claude.ai > Organization settings > API に移動します。

Public API access を有効化し、Analytics API key を作成します。

read:analytics スコープを持つキーを作成し、表示された secret を secrets manager に保存します。(Claude)

x-api-key ヘッダーにキーを渡して、/v1/organizations/analytics/ 配下の endpoint を呼び出します。(Claude)

Analytics API で取得できるデータには、ユーザー別の日次 activity、DAU/WAU/MAU や seat count などの summary、Project / Skill / Connector usage、ユーザー別・組織別の token usage / cost report などがあります。(Claude)

データ鮮度には注意が必要です。Claude Enterprise Analytics API のデータは 2026年1月1日以降 の日付が対象です。Engagement / adoption endpoints は指定日の per-day snapshot で、翌日10:00 UTC に集計され、その3日後にクエリ可能になります。Cost / usage endpoints は通常4時間以内、最大24時間で利用可能になりますが、遅延イベントや照合処理により最大30日間修正される可能性があります。請求照合レベルの安定値が必要な場合は、少なくとも30日以上前の日付を問い合わせるのが公式推奨です。(Claude)

### 6-2. Compliance API

用途は、セキュリティ、法務、監査、eDiscovery、SIEM連携です。Compliance API は、Activity Feed、ユーザー・ロール・グループ、組織設定、claude.ai 組織の chats / files / projects などへのプログラムアクセスを提供します。(Claude)

手順は次の通りです。

Primary Owner で Claude にサインインします。Claude Enterprise primary owners は claude.ai で Compliance API を有効化できます。(Claude)

claude.ai > Organization settings > API に移動します。

Compliance API を有効化します。

Keys セクションで Create key を実行します。

必要最小限のスコープを選択します。Activity Feed のみなら read:compliance_activities、チャット・ファイル・プロジェクト等を読む場合は read:compliance_user_data などが必要です。公式ドキュメントは、必要最小限のスコープを選ぶこと、読み取り用と削除用を分けることを推奨しています。(Claude)

表示された secret key を secrets manager に保存します。secret は一度しか表示されません。(Claude)

Activity Feed の呼び出し例は以下です。

```text
curl --fail-with-body -sS \
"https://api.anthropic.com/v1/compliance/activities?limit=1" \
--header "x-api-key: $ANTHROPIC_COMPLIANCE_ACCESS_KEY"
```

Compliance API の各 endpoint は https://api.anthropic.com の /v1/compliance/* 配下にあり、x-api-key ヘッダーで認証します。Activity Feed は read:compliance_activities スコープを持つキーで利用でき、それ以外の endpoint には関連スコープを持つ Compliance Access Key が必要です。(Claude)

監査ログ CSV export は Compliance API より狭い用途の機能で、lookback window が制限され、CSV download のみで、チャット・ファイル・プロジェクト本文にはアクセスできません。継続的なプログラム利用では Compliance API を標準化するよう公式に説明されています。(Claude)

## 7. 間違えやすい点

Claude Enterprise、つまり claude.ai 側の Enterprise parent organization は Claude Console に表示されず、Admin API key も持ちません。そのため、Claude Enterprise の cost / usage data を API で取る場合は Analytics API key が経路になります。(Claude)

一方で、Claude Console / Claude Platform の API 利用量・コストを見る場合は、Console の Usage / Cost 画面や Usage and Cost Admin API が対象です。Console の Usage page では model、month/day、API key などで絞り込み、表示データを CSV export できます。Cost page も export button から CSV をダウンロードできます。(Claude ヘルプセンター)

## 導入案件向けの推奨運用

月次の社内報告なら、まず Analytics > Spend > Export Spend Report を標準手順にするのが最も簡単です。監査・証跡用途なら Export logs、会話データを含む組織データ確認なら Export Data、継続連携やSIEM/BI連携なら Analytics API / Compliance API を使い分けるのが妥当です。

公式情報で確認できなかった点は、CSVの文字コード、ファイル名規則、organization data export の詳細スキーマ、Analytics 画面の Spend 以外の全指標に対する一括CSV出力可否です。これらは 不明 です。
