**Entra ID 側の詳細手順（全員SSO必須／管理者のみMFA）**

- 1) **グループ準備**
  - `OpenAI-Users`（SSO対象の全ユーザー）
  - `OpenAI-Admins`（管理者のみ。`OpenAI-Users`の部分集合）

- 2) **SAMLエンタープライズアプリ作成**
  - Entra 管理センター →「エンタープライズ アプリケーション」→「新しいアプリケーション」
  - 「独自のアプリケーションを作成」→「ギャラリーにないアプリを統合（Non-gallery）」で作成
  - 作成したアプリ →「シングル サインオン」→「SAML」

- 3) **基本 SAML 構成（OpenAIで提示される値を貼り付け）**
  - OpenAI Identity のSSO設定ウィザードで表示される **`Entity ID`** と **`SSO URL`** を、Entra の「基本 SAML 構成」に設定（通常は `Identifier (Entity ID)` と `Reply URL (ACS)` に相当） ([help.openai.com](https://help.openai.com/ja-jp/articles/10472980-chatgpt-%E3%81%A7-sso-%E3%82%92%E6%9C%89%E5%8A%B9%E3%81%AB%E3%81%99%E3%82%8B))
  - （任意）IdP起点ログイン（My Appsのタイル）を使うなら、Entra 側の `Sign on URL` に OpenAI が提示するタイルURLを設定（形式は下記） ([help.openai.com](https://help.openai.com/ja-jp/articles/10468051-sso-%E3%81%AE%E6%A6%82%E8%A6%81?utm_source=openai))
    - `https://chatgpt.com/auth/login?sso=true&connection=conn_...`
    - `https://platform.openai.com/enterprise/conn_.../login`
  - 注意：OpenAI側でSSO接続をリセットするとタイルURLが変わるため、Entra の `Sign on URL` も更新が必要です。 ([help.openai.com](https://help.openai.com/ja-jp/articles/10472980-chatgpt-%E3%81%A7-sso-%E3%82%92%E6%9C%89%E5%8A%B9%E3%81%AB%E3%81%99%E3%82%8B))

- 4) **クレーム/属性（重要）**
  - **メールアドレスは必須**、名/姓は任意だが推奨（どの属性をどのクレームに出すかは、OpenAIウィザードの推奨に合わせる）。 ([help.openai.com](https://help.openai.com/ja-jp/articles/10472980-chatgpt-%E3%81%A7-sso-%E3%82%92%E6%9C%89%E5%8A%B9%E3%81%AB%E3%81%99%E3%82%8B))
  - **SAML応答/アサーションを暗号化しない**（OpenAIは復号に非対応）。 ([help.openai.com](https://help.openai.com/ja-jp/articles/10472980-chatgpt-%E3%81%A7-sso-%E3%82%92%E6%9C%89%E5%8A%B9%E3%81%AB%E3%81%99%E3%82%8B))
  - メールが複数出ないようにし、可能なら変更されにくいメール（例：UPN）を使う。 ([help.openai.com](https://help.openai.com/ja-jp/articles/10472980-chatgpt-%E3%81%A7-sso-%E3%82%92%E6%9C%89%E5%8A%B9%E3%81%AB%E3%81%99%E3%82%8B))

- 5) **EntraのメタデータをOpenAIへ（Dynamic推奨）**
  - Entra のSAML設定画面で「フェデレーション メタデータ URL（またはメタデータXML）」を取得
  - OpenAI Identity ウィザードで `Dynamic Configuration` を選び、そのメタデータURLを貼り付け（推奨）。 ([help.openai.com](https://help.openai.com/ja-jp/articles/10472980-chatgpt-%E3%81%A7-sso-%E3%82%92%E6%9C%89%E5%8A%B9%E3%81%AB%E3%81%99%E3%82%8B))
  - `Manual Configuration` の場合は、Entra から `SSO URL` / `Issuer` / `x.509 証明書` を取り出してOpenAI側に入力。 ([help.openai.com](https://help.openai.com/ja-jp/articles/10472980-chatgpt-%E3%81%A7-sso-%E3%82%92%E6%9C%89%E5%8A%B9%E3%81%AB%E3%81%99%E3%82%8B))

- 6) **ユーザー/グループ割り当て（SSO必須運用の肝）**
  - エンタープライズ アプリ →「ユーザーとグループ」→ `OpenAI-Users` を割り当て（管理者も必ず含める）
  - OpenAI側ウィザードでも「グループでアクセス付与」の案内が出るので整合させる。 ([help.openai.com](https://help.openai.com/ja-jp/articles/10472980-chatgpt-%E3%81%A7-sso-%E3%82%92%E6%9C%89%E5%8A%B9%E3%81%AB%E3%81%99%E3%82%8B))

- 7) **管理者だけMFA（条件付きアクセス）**
  - Entra 管理センター →「条件付きアクセス」→「新規ポリシー」
  - 対象ユーザー：`OpenAI-Admins` のみ
  - 対象クラウドアプリ：上で作成した OpenAI のエンタープライズアプリ
  - 付与：`多要素認証を必須` を有効化
  - 既存ポリシーで「全ユーザーにMFA必須」が当たっていないことも確認（当たっていると非管理者にもMFAがかかります）

- 8) **テスト**
  - OpenAI Identity ウィザードの `Test Single Sign-On` で接続テスト。 ([help.openai.com](https://help.openai.com/ja-jp/articles/10472980-chatgpt-%E3%81%A7-sso-%E3%82%92%E6%9C%89%E5%8A%B9%E3%81%AB%E3%81%99%E3%82%8B))
  - 非管理者（MFAなし）／管理者（MFAあり）でログイン確認。

可能です。ただし「OpenAI側で Owner/Admin だけ（またはワークスペース単位で）MFA を必須化」は現時点ではできません。([help.openai.com](https://help.openai.com/en/articles/7967234-enabling-multi-factor-authentication-mfa-with-openai))

- **Owner/Admin のアカウントでMFA**
  - OpenAIのMFAは各ユーザーのOpenAIアカウント単位で有効化でき、ChatGPT等の全OpenAIサービスに適用されます。([help.openai.com](https://help.openai.com/en/articles/7967234-enabling-multi-factor-authentication-mfa-with-openai))
  - ただし、MFAをワークスペース／組織単位で「強制」する機能は未対応です（＝Owner/AdminだけをOpenAI側設定で必須化も不可）。([help.openai.com](https://help.openai.com/en/articles/7967234-enabling-multi-factor-authentication-mfa-with-openai))
  - SSOはIdPがユーザーを認証する仕組みなので、SSOログイン時にMFAを必須にしたい場合はIdP（Entra ID）側での要件化になります。([help.openai.com](https://help.openai.com/en/articles/10468051-sso-overview))

- **Global Admin Console（`admin.openai.com`）も SSO＋MFA できるか**
  - Global Admin Console には「Admin Portal SSO Settings」があり、`admin.openai.com`へのログイン方法をSSO（Required/Optional/Off）で制御できます。([help.openai.com](https://help.openai.com/en/articles/12289294-global-admin-console))
  - よってIdP側でMFAを要求していれば、Global Admin Console のログインも SSO＋MFA になります。([help.openai.com](https://help.openai.com/en/articles/10468051-sso-overview))
  - なお OpenAI は、設定誤りによるロックアウト回避のため、セットアップ中は Admin Portal 側SSOを Optional に保つことを推奨しています。([help.openai.com](https://help.openai.com/en/articles/10472980-enabling-sso-on-chatgpt))

以下は **Entra ID（Azure AD）でSSOし、IdP側MFAも必須にする** ための、作業場所つきの手順です（参照はOpenAI公式ヘルプのみ）。 ([help.openai.com](https://help.openai.com/ja-jp/articles/10468051-sso-overview))

## 0) 事前に決める（設計）
- **対象ドメイン**（例：`example.com`）。※サブドメインはトップレベルとは別に検証が必要です。 ([help.openai.com](https://help.openai.com/ja-jp/articles/10468051-sso-overview))  
- **SSO適用範囲**：ChatGPTだけ／API Platformも／`admin.openai.com`（Global Admin Console）も。ChatGPTとAPI Platformは“有効化スイッチ”が別です。 ([help.openai.com](https://help.openai.com/ja-jp/articles/10468051-sso-overview))  
- **Entra側の対象グループ**（誰にSSO＋MFAをかけるか）

---

## 1) OpenAI側（Global Admin Console / OpenAI Identity）でドメイン検証
**作業場所：OpenAI（Global Admin Console）**
1. `admin.openai.com` → **「ID とアクセス」** → **「ドメイン」** で `+ Add Domain`。 ([help.openai.com](https://help.openai.com/ja-jp/articles/12289294-global-admin-console))  
2. 表示される値で **DNSにTXTレコード** を追加 → OpenAI画面で `Check` → `Verified` を確認。 ([help.openai.com](https://help.openai.com/ja-jp/articles/9534785-configuring-sso))  

補足：
- ドメイン検証はSSOの前提です。 ([help.openai.com](https://help.openai.com/ja-jp/articles/9534785-configuring-sso))  
- 検証直後は既定で「未マッピング」になり、**マッピングしない限りSSOは実質非アクティブ** です（あとで設定します）。 ([help.openai.com](https://help.openai.com/ja-jp/articles/12289294-global-admin-console))  

---

## 2) OpenAI側（OpenAI Identity）でSSO接続作成ウィザード開始
**作業場所：OpenAI（OpenAI Identity）**
1. `admin.openai.com` の **OpenAI Identity（`admin.openai.com/identity`）** に移動し、`Set up SSO` を開始。 ([help.openai.com](https://help.openai.com/ja-jp/articles/9534785-configuring-sso))  
2. **IdPとして Entra ID** を選択し、ウィザードを進めます。 ([help.openai.com](https://help.openai.com/ja-jp/articles/9534785-configuring-sso))  

重要（ロックアウト対策）：
- 設定中は **ログイン済みブラウザを2つ（通常＋シークレット）** 開いたままにし、**管理者ポータルSSOはOptional推奨** です。 ([help.openai.com](https://help.openai.com/ja-jp/articles/9534785-configuring-sso))  

---

## 3) Azure側（Entra ID）でSAMLアプリ作成＋MFA必須化
**作業場所：Azure（Entra ID）**
1. OpenAIのSSOウィザードが提示する **URL（SSO URL / Entity ID など）** を、Entraの **SAMLアプリ設定** に転記（画面内で案内されます）。 ([help.openai.com](https://help.openai.com/ja-jp/articles/9534785-configuring-sso))  
2. **属性（クレーム）マッピング**：SAMLレスポンスに **メールアドレス必須**、名/姓は任意（推奨）。マッピングは **ウィザードのIdP別推奨** に合わせます。 ([help.openai.com](https://help.openai.com/ja-jp/articles/9534785-configuring-sso))  
3. **SAMLレスポンス/アサーションを暗号化しない**（OpenAIはSAMLレスポンス復号に未対応）。 ([help.openai.com](https://help.openai.com/ja-jp/articles/9534785-configuring-sso))  
4. **アプリへの割当**：SSO対象ユーザー/グループを割り当て（OpenAIウィザードでも「グループでアクセス付与」の流れが出ます）。 ([help.openai.com](https://help.openai.com/ja-jp/articles/9534785-configuring-sso))  
5. **MFAを必須化**：Entra側で、このアプリへのサインインに **MFAを要求** するポリシー（例：条件付きアクセス等）を作成・適用します（MFAの強制はIdP側で実施する想定になります）。 ([help.openai.com](https://help.openai.com/ja-jp/articles/10468051-sso-overview))  

---

## 4) OpenAI側（OpenAI Identity）へIdPメタデータ登録 → テスト
**作業場所：OpenAI（OpenAI Identity）**
1. **IdPメタデータ設定**  
   - 推奨：**Dynamic Configuration**（Entraで取得した **メタデータURL** を入力） ([help.openai.com](https://help.openai.com/ja-jp/articles/9534785-configuring-sso))  
   - 代替：**Manual Configuration**（SSO URL / IdP Issuer / x.509証明書 を入力） ([help.openai.com](https://help.openai.com/ja-jp/articles/9534785-configuring-sso))  
2. （任意）**IdP起点ログイン（タイル）** を作る場合：OpenAIが提示する `Sign on URL`（ChatGPT/Platformそれぞれ）をEntra側の該当欄に設定。 ([help.openai.com](https://help.openai.com/ja-jp/articles/9534785-configuring-sso))  
   - 接続をリセットするとURLが変わるので、その都度更新が必要です。 ([help.openai.com](https://help.openai.com/ja-jp/articles/9534785-configuring-sso))  
3. ウィザードの **「Test Single Sign-On」** で `Continue to sign-in` を実行し、SSOが成功することを確認。 ([help.openai.com](https://help.openai.com/ja-jp/articles/9534785-configuring-sso))  

---

## 5) OpenAI側（Global Admin Console）で「適用」＝ドメインマッピング＋SSO有効化
**作業場所：OpenAI（Global Admin Console）**
1. `admin.openai.com` → **「ID とアクセス」→「ドメインの対象条件とマッピング」** で、検証済みドメインを **対象のChatGPTワークスペース** にマッピング。 ([help.openai.com](https://help.openai.com/ja-jp/articles/12289294-global-admin-console))  
   - **API Platformはドメインベース** なので、影響範囲に注意（テナント外を含むAPI組織にもグローバルに効き得ます）。 ([help.openai.com](https://help.openai.com/ja-jp/articles/12289294-global-admin-console))  
2. **「シングルサインオン（SSO）」** で、適用先ごとに `Optional →（問題なければ）Required` に切り替え。 ([help.openai.com](https://help.openai.com/ja-jp/articles/12289294-global-admin-console))  
3. **「管理者ポータルのSSO設定」**（`admin.openai.com` へのログイン方式）も同様に `Optional → Required` へ。 ([help.openai.com](https://help.openai.com/ja-jp/articles/12289294-global-admin-console))  

---

## 6) ChatGPT側（ワークスペース）でユーザー招待/プロビジョニング確認
**作業場所：ChatGPT（ワークスペース設定）**
- SSOは“認証”で、ユーザーが使うには **ワークスペースへの招待（またはSCIM等）** が別途必要です。 ([help.openai.com](https://help.openai.com/ja-jp/articles/10468051-sso-overview))  
- ワークスペース設定UIは更新されており、「アイデンティティとプロビジョニング」等は **「アイデンティティとアクセス」タブに統合** されています。 ([help.openai.com](https://help.openai.com/ja-jp/articles/8411955-what-workspace-settings-can-i-control-for-my-workspace))  

---

## 7) 追加の安全策（任意）：OpenAIアカウントのMFA
**作業場所：ChatGPT / API Platform（個人アカウント設定）**
- 管理者がパスワード等でログインできる経路を残す運用（SSO Optional期間など）なら、各管理者アカウントで **OpenAI側MFA** を有効化しておくと補強になります。 ([help.openai.com](https://help.openai.com/ja-jp/articles/7967234-enabling-or-disabling-multi-factor-authentication-mfa?utm_source=openai))  
- ただし **OpenAI側MFAをワークスペース/組織単位で強制はできません**。 ([help.openai.com](https://help.openai.com/ja-jp/articles/7967234-enabling-or-disabling-multi-factor-authentication-mfa?utm_source=openai))  

必要なら、あなたの想定（1) ChatGPTだけSSO必須、2) Platformも必須、3) `admin.openai.com` も必須）を教えてください。影響範囲（特にPlatformのドメインベース）に合わせて、切り替え順序（Optional→Required）を具体化します。 ([help.openai.com](https://help.openai.com/ja-jp/articles/10468051-sso-overview))
