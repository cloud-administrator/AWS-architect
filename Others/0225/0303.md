**Entra ID 側の詳細手順（全員SSO必須／管理者のみMFA）**

- 1) **グループ準備**
  - `OpenAI-Users`（SSO対象の全ユーザー）
  - `OpenAI-Admins`（管理者のみ。`OpenAI-Users`の部分集合）

- 2) **SAMLエンタープライズアプリ作成**
  - Entra 管理センター →「エンタープライズ アプリケーション」→「新しいアプリケーション」
  - 「独自のアプリケーションを作成」→「ギャラリーにないアプリを統合（Non-gallery）」で作成
  - 作成したアプリ →「シングル サインオン」→「SAML」

- 3) **基本 SAML 構成（OpenAIで提示される値を貼り付け）**
  - OpenAI Identity のSSO設定ウィザードで表示される **`Entity ID`** と **`SSO URL`** を、Entra の「基本 SAML 構成」に設定（通常は `Identifier (Entity ID)` と `Reply URL (ACS)` に相当） ([help.openai.com](https://help.openai.com/ja-jp/articles/10472980-chatgpt-%E3%81%A7-sso-%E3%82%92%E6%9C%89%E5%8A%B9%E3%81%AB%E3%81%99%E3%82%8B))
  - （任意）IdP起点ログイン（My Appsのタイル）を使うなら、Entra 側の `Sign on URL` に OpenAI が提示するタイルURLを設定（形式は下記） ([help.openai.com](https://help.openai.com/ja-jp/articles/10468051-sso-%E3%81%AE%E6%A6%82%E8%A6%81?utm_source=openai))
    - `https://chatgpt.com/auth/login?sso=true&connection=conn_...`
    - `https://platform.openai.com/enterprise/conn_.../login`
  - 注意：OpenAI側でSSO接続をリセットするとタイルURLが変わるため、Entra の `Sign on URL` も更新が必要です。 ([help.openai.com](https://help.openai.com/ja-jp/articles/10472980-chatgpt-%E3%81%A7-sso-%E3%82%92%E6%9C%89%E5%8A%B9%E3%81%AB%E3%81%99%E3%82%8B))

- 4) **クレーム/属性（重要）**
  - **メールアドレスは必須**、名/姓は任意だが推奨（どの属性をどのクレームに出すかは、OpenAIウィザードの推奨に合わせる）。 ([help.openai.com](https://help.openai.com/ja-jp/articles/10472980-chatgpt-%E3%81%A7-sso-%E3%82%92%E6%9C%89%E5%8A%B9%E3%81%AB%E3%81%99%E3%82%8B))
  - **SAML応答/アサーションを暗号化しない**（OpenAIは復号に非対応）。 ([help.openai.com](https://help.openai.com/ja-jp/articles/10472980-chatgpt-%E3%81%A7-sso-%E3%82%92%E6%9C%89%E5%8A%B9%E3%81%AB%E3%81%99%E3%82%8B))
  - メールが複数出ないようにし、可能なら変更されにくいメール（例：UPN）を使う。 ([help.openai.com](https://help.openai.com/ja-jp/articles/10472980-chatgpt-%E3%81%A7-sso-%E3%82%92%E6%9C%89%E5%8A%B9%E3%81%AB%E3%81%99%E3%82%8B))

- 5) **EntraのメタデータをOpenAIへ（Dynamic推奨）**
  - Entra のSAML設定画面で「フェデレーション メタデータ URL（またはメタデータXML）」を取得
  - OpenAI Identity ウィザードで `Dynamic Configuration` を選び、そのメタデータURLを貼り付け（推奨）。 ([help.openai.com](https://help.openai.com/ja-jp/articles/10472980-chatgpt-%E3%81%A7-sso-%E3%82%92%E6%9C%89%E5%8A%B9%E3%81%AB%E3%81%99%E3%82%8B))
  - `Manual Configuration` の場合は、Entra から `SSO URL` / `Issuer` / `x.509 証明書` を取り出してOpenAI側に入力。 ([help.openai.com](https://help.openai.com/ja-jp/articles/10472980-chatgpt-%E3%81%A7-sso-%E3%82%92%E6%9C%89%E5%8A%B9%E3%81%AB%E3%81%99%E3%82%8B))

- 6) **ユーザー/グループ割り当て（SSO必須運用の肝）**
  - エンタープライズ アプリ →「ユーザーとグループ」→ `OpenAI-Users` を割り当て（管理者も必ず含める）
  - OpenAI側ウィザードでも「グループでアクセス付与」の案内が出るので整合させる。 ([help.openai.com](https://help.openai.com/ja-jp/articles/10472980-chatgpt-%E3%81%A7-sso-%E3%82%92%E6%9C%89%E5%8A%B9%E3%81%AB%E3%81%99%E3%82%8B))

- 7) **管理者だけMFA（条件付きアクセス）**
  - Entra 管理センター →「条件付きアクセス」→「新規ポリシー」
  - 対象ユーザー：`OpenAI-Admins` のみ
  - 対象クラウドアプリ：上で作成した OpenAI のエンタープライズアプリ
  - 付与：`多要素認証を必須` を有効化
  - 既存ポリシーで「全ユーザーにMFA必須」が当たっていないことも確認（当たっていると非管理者にもMFAがかかります）

- 8) **テスト**
  - OpenAI Identity ウィザードの `Test Single Sign-On` で接続テスト。 ([help.openai.com](https://help.openai.com/ja-jp/articles/10472980-chatgpt-%E3%81%A7-sso-%E3%82%92%E6%9C%89%E5%8A%B9%E3%81%AB%E3%81%99%E3%82%8B))
  - 非管理者（MFAなし）／管理者（MFAあり）でログイン確認。
